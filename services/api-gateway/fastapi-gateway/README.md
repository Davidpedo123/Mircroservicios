La API Gateway actúa como intermediario entre los clientes y los microservicios, proporcionando diversas funcionalidades clave como:

Autenticación y Autorización: Manejo de usuarios y validación de tokens a través de rutas definidas en routes/auth.
Balanceo de Carga: Distribuye el tráfico entre múltiples instancias de microservicios en routes/load_balancer.
SSL y Seguridad: Implementación de SSL para comunicaciones seguras a través de routes/ssl.
Caché: Mejoras en el rendimiento mediante la implementación de caché en routes/cache.
Gestión de Productos y Usuarios: Rutas dedicadas para microservicios de productos y usuarios en routes/products y routes/users.
